Direction A — what to notice

  • Wordmark is top-left, small and quiet.
  • Input has a visible 1 px border that anchors it as a deliberate object.
  • Drag handle (⋮⋮) is visible on every row, on every viewport — affordance is self-evident.
  • Completed items stay where they are; sort order is stable.
  • Failed item shows the warning glyph + tinted background + inline retry, with no modal.
  • Diagonal hairline runs across the completed item (-12°), behind the text — your "cracked" idea, quieted.
  • Desktop adds barely-perceptible shadow under each card; mobile uses border alone.

Direction B — what to notice

  • Input has no border by default — it visually blends with the page until hovered or focused.
  • Drag handles are hidden on desktop until you hover a row (try it). They remain always visible on mobile, where there's no hover.
  • Restraint is more pronounced; the page is more uniform.
  • Trade-off: drag-to-reorder is less discoverable on desktop. Lina might never find it without hovering a row.

Direction C — what to notice

  • Wordmark is centered above the input.
  • Completed items are auto-grouped into a separate section below active items, with a soft dashed divider.
  • Trade-off: when a user toggles an item complete, it moves on its own from the active section to the completed section — that's surprise / non-direct motion, which violates Step 4 Principle 6 ("Motion only where the user acted"). Even rendered statically, the item disappears from where the user expected it to stay.
